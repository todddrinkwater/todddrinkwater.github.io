Design and Structural Features

Simpilicity - "In a study by Google in August of 2012, researchers found that not only will users judge websites as beautiful or not within 1/50th – 1/20th of a second, but also that “visually complex” websites are consistently rated as less beautiful than their simpler counterparts." (Source: http://conversionxl.com/why-simple-websites-are-scientifically-better/)

The other side to this is cognitive fluency. This is the idea that things on the website appear when you expect them too and the website behaves like you expect it would. (Conversion XL).

The simpler the website, the more appealing. Simplicity makes the website more appealing and there are less distractions to take away from your key message.

Layering - This concept gives a bit more depth to a website, and can be effective if you wish to display a key image in the background. Ideas of use could be for websites wanting to display prominent images to help sell and idea or product, or to help build a theme for a website.

Sliding Images with Optional Click Button - Apple's website displays this well. It displays big, high-definition images of the products it wishes to sell. For companies like Apple, which only have a small product line, this concept maximises the full width of the page to promote its products. The option click button allows users to scroll backwards or forwards at their own pace.

Understanding HTML, CSS, and the DOM model

HTML

HTML is the backbone of your webpage. Think of it in terms of Cycling. If you were to build a bike from scratch, you might think that your bike is going to have two wheels, a frame, gears, and really imporantly, brakes. This would be the structure of a bike that works.

CSS

CSS is a language utilized for styling a webpage. Although you might know what your Bike needs to have to work, you now have the option of using different materials on your bike that would suit what you intend to use it for. For example, a road bike may have thin light wheels, a carbon fibre frame and maybe some flames down the side if you're feeling kick-ass. A mountain bike however might come with a titanium, suspension frame and thicker tyres, to handle the pressure of off-road biking.

The DOM Model

The DOM model is a programming interface that is built into the application you use to browse the internet (e.g. Google Chrome). This interface acts like an interpreter or translator. It takes the code that you give the application (e.g. HTML) and interprets it following a set of rules that decided on how the code should be interpreted. Within the DOM, different parts of code are given different hierachies which helps decide when and how the code will be interpreted.

Boxes, boxes and more boxes

What is meant by Boxifying Design?

Boxifying design is the concept of viewing your webpage as a series of boxes. When looking at a webpage, you should be able to break it up into a series of boxes, even the rounder images are grouped into boxes. Using this concept, it makes it easy for developers to arrange the content of a webpage effectively.

What is the box model?

The Box Model is a design concept that demonstrates what is within and outside of the box that surrounds every HTML element located on a page. Within this model, it show the content, surrounded by the padding, then border, and finally the margin. This model is helpful when trying to understand how space works in between elements.
